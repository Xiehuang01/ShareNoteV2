# ShareNote V2 - 前端架构文档

## 项目概述

ShareNote V2 是一个基于 Vue 3 的现代化笔记管理系统，支持 Markdown 编辑、AI 辅助、小组协作等功能。

## 技术栈

### 核心框架
- **Vue 3** - 渐进式 JavaScript 框架
- **Vite** - 下一代前端构建工具
- **Pinia** - Vue 3 官方状态管理库
- **Vue Router** - 官方路由管理器

### UI 框架
- **Element Plus** - 基于 Vue 3 的组件库
- **SCSS** - CSS 预处理器

### 功能库
- **Axios** - HTTP 客户端
- **diff-match-patch** - 文本差异对比库
- **Marked** - Markdown 解析器
- **highlight.js** - 代码高亮

## 路由配置

### 路由表

```javascript
[
  {
    path: '/login',
    component: LoginPage,
    meta: { requiresAuth: false }
  },
  {
    path: '/forgetpassword',
    component: ForgetPassword,
    meta: { requiresAuth: false }
  },
  {
    path: '/',
    component: LayoutPage,
    redirect: '/main/notespage',
    meta: { requiresAuth: true },
    children: [
      {
        path: '/main/notespage',
        component: NotesPage
      },
      {
        path: '/main/settings',
        component: SettingsPage
      },
      {
        path: '/main/profile',
        component: ProfilesPage
      }
    ]
  }
]
```

### 路由守卫

```javascript
router.beforeEach((to) => {
  const userStore = useUserStore()
  // 未登录且访问需要认证的页面，重定向到登录页
  if (!userStore.token && to.path !== '/login' && to.path !== '/forgetpassword') {
    return '/login'
  }
})
```

## HTTP 请求封装

### Axios 实例配置

```javascript
const instance = axios.create({
  baseURL: 'http://localhost:3000',
  timeout: 10000
})
```

### 请求拦截器

```javascript
instance.interceptors.request.use(
  (config) => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = userStore.token
    }
    return config
  },
  (err) => Promise.reject(err)
)
```

### 响应拦截器

```javascript
instance.interceptors.response.use(
  (res) => res.data,
  (err) => Promise.reject(err)
)
```

## 状态管理

### Pinia Store 结构

```javascript
export const useUserStore = defineStore('sharenote-user', () => {
  // 状态
  const token = ref('')
  const userInfo = ref({})
  const userNotesList = ref([])
  const selectedNote = ref({})
  const chooseNote = ref(null)
  const isEditMode = ref(false)
  const currentGroup = ref(null)
  const currentMemberId = ref(null)
  const groupMembers = ref([])
  const aiConfig = ref(null)

  // 方法
  const setToken = (newToken) => { token.value = newToken }
  const removeToken = () => { token.value = '' }
  // ... 其他方法

  return {
    token, setToken, removeToken,
    // ... 其他状态和方法
  }
}, {
  persist: {
    key: 'sharenote-user',
    storage: localStorage,
    paths: ['token', 'userInfo', 'userNotesList', 'selectedNote', 'chooseNote', 'currentGroup', 'currentMemberId', 'groupMembers', 'aiConfig']
  }
})
```

### 持久化策略

使用 `pinia-plugin-persistedstate` 插件实现状态持久化：
- 存储位置：`localStorage`
- 存储键：`sharenote-user`
- 持久化字段：token、userInfo、userNotesList、selectedNote、chooseNote、currentGroup、currentMemberId、groupMembers、aiConfig
- 非持久化字段：isEditMode（每次刷新重置）

## 错误处理

### 全局错误处理
```javascript
// 请求错误
instance.interceptors.response.use(
  (res) => res.data,
  (err) => {
    if (err.response?.status === 401) {
      // Token 过期，跳转登录
      router.push('/login')
    }
    return Promise.reject(err)
  }
)
```

### 组件错误处理
```javascript
try {
  const res = await apiCall()
  if (res.status === 'success') {
    ElMessage.success(res.message)
  } else {
    ElMessage.error(res.message)
  }
} catch (error) {
  ElMessage.error(error.response?.data?.message || '操作失败')
}
```

## 常见问题

### Q: 如何配置 AI 功能？
A: 进入设置页面，填写 SiliconFlow API 密钥和选择模型，点击测试连接后保存。

### Q: 为什么上传的图片没有显示？
A: 确保 Markdown 文件和图片一起上传，系统会自动关联图片路径。

### Q: 如何切换小组？
A: 点击顶部导航栏的"切换小组"按钮，选择要切换的小组。

### Q: 忘记密码怎么办？
A: 点击登录页面的"忘记密码"，通过邮箱验证码重置密码。
