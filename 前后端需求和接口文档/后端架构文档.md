# ShareNote V2 - 后端架构文档

## 项目概述

ShareNote V2 后端是一个基于 Node.js + Express 的 RESTful API 服务，提供用户认证、笔记管理、小组协作等功能。

## 技术栈

### 核心框架
- **Node.js** - JavaScript 运行时
- **Express** - Web 应用框架
- **MySQL** - 关系型数据库

### 主要依赖
- **jsonwebtoken** - JWT Token 生成与验证
- **bcryptjs** - 密码加密
- **multer** - 文件上传处理
- **sharp** - 图片处理
- **nodemailer** - 邮件发送
- **redis** - 缓存和会话管理
- **dayjs** - 日期时间处理
- **cors** - 跨域资源共享

## 中间件

### JWT 认证中间件 (auth.js)

```javascript
export const checkAuth = (req, res, next) => {
  const token = req.headers['authorization']
  if (!token) {
    return res.status(401).json({
      status: 'fail',
      message: '未查找到用户的token权限不够，请登录后再次尝试'
    })
  }
  try {
    const resJwtVerify = jwt.verify(token, SECRET_KEY)
    req.userId = resJwtVerify.id
    next()
  } catch {
    return res.status(401).json({
      status: 'fail',
      message: 'token无效或已过期，请重新登录'
    })
  }
}
```

### 使用方式
```javascript
router.get('/protected', checkAuth, async (req, res) => {
  const userId = req.userId // 从中间件获取用户ID
  // 处理请求...
})
```

## 安全措施

### 密码安全
- bcrypt 加密（加盐强度：10）
- 密码不可逆加密
- 登录时使用 compareSync 验证

### Token 安全
- JWT Token 认证
- Token 存储在请求头
- Token 过期自动失效
- 密钥：`aqgy1213812138`

### 文件安全
- 文件类型验证
- 文件大小限制
- 文件路径验证
- 权限验证（只能操作自己的文件）

### SQL 注入防护
- 使用参数化查询
- 所有数据库操作使用 `pool.execute(sql, params)`

## 性能优化

### 数据库优化
- 使用连接池（最大10个连接）
- 添加索引（用户名、邮箱、文件ID等）
- 软删除（isActive 字段）

### 文件处理优化
- 图片自动压缩
- 头像裁剪为固定尺寸
- 临时文件自动清理

### 缓存策略
- Redis 缓存验证码
- 验证码5分钟有效期
- 验证后自动删除

## 常见问题

### Q: 如何重置数据库？
A: 运行 `node database/migrate.js` 重新创建表结构。

### Q: 邮件发送失败怎么办？
A: 检查 SMTP 配置和授权码是否正确。

### Q: 文件上传失败？
A: 检查 upload 目录权限和磁盘空间。

### Q: Token 验证失败？
A: 检查 Token 是否过期或密钥是否正确。
