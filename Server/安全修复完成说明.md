## 🔧 配置步骤

### 1. 配置环境变量

编辑 `server/.env` 文件，填入真实的配置信息：

```env
# 数据库配置
DB_HOST=103.194.106.239
DB_PORT=3306
DB_USER=serveruser
DB_PASSWORD=serveruser12138
DB_NAME=sharenotev2

# JWT 配置（请修改为更安全的密钥）
JWT_SECRET=your-super-secret-jwt-key-change-this-to-random-string
JWT_EXPIRES_IN=7d

# 邮件配置（填入真实的授权码）
EMAIL_HOST=smtp.163.com
EMAIL_PORT=465
EMAIL_USER=xiehuang_top@163.com
EMAIL_PASSWORD=your-email-auth-code-here

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379

# 服务器配置
PORT=3000
NODE_ENV=development

# CORS 配置（添加你的前端地址）
ALLOWED_ORIGINS=http://localhost:5173
```

### 2. 生成安全的 JWT 密钥

```bash
# 使用 Node.js 生成随机密钥
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

将生成的密钥替换 `.env` 中的 `JWT_SECRET`

### 3. 启动服务器

```bash
cd server
node server.js
```

你应该看到类似的输出：

```
✓ 数据库连接成功
✓ Redis 连接成功
✓ 邮件服务连接成功
=================================
✓ ShareNote Server 启动成功
✓ 运行端口: 3000
✓ 环境: development
✓ 允许的来源: http://localhost:5173
=================================
```

## ⚠️ 重要提醒

### 1. 不要提交 .env 文件到 Git

已创建 `.gitignore` 文件，确保 `.env` 不会被提交。

### 2. 生产环境配置

生产环境需要额外配置：

```env
NODE_ENV=production
ALLOWED_ORIGINS=https://yourdomain.com,https://www.yourdomain.com
```

### 3. 更改默认密钥

**必须**更改以下默认值：
- `JWT_SECRET` - 使用随机生成的密钥
- `EMAIL_PASSWORD` - 填入真实的邮箱授权码

### 4. 数据库安全

建议：
- 不要使用公网 IP 直接连接数据库
- 使用 VPN 或 SSH 隧道
- 定期更换数据库密码
- 限制数据库访问 IP

## 📞 遇到问题？

### 常见问题

**Q: 启动时提示数据库连接失败？**
A: 检查 `.env` 中的数据库配置是否正确

**Q: CORS 错误？**
A: 确保前端地址已添加到 `ALLOWED_ORIGINS`

**Q: Token 验证失败？**
A: 检查 `JWT_SECRET` 是否一致

**Q: 邮件发送失败？**
A: 检查邮箱授权码是否正确
