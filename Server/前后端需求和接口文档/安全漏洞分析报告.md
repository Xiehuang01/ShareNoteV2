## 🔴 高危漏洞

### 1. JWT 密钥硬编码（严重）

**位置**：
- `server/routers/login.js:45`
- `server/middlewares/auth.js:2`

**问题描述**：
```javascript
const SECRET_KEY = 'aqgy1213812138'  // 硬编码在代码中
```

**风险等级**：🔴 严重

**影响**：
- 密钥泄露后攻击者可以伪造任意用户的 Token
- 可以绕过所有认证机制
- 无法轻易更换密钥（需要修改代码）

**修复建议**：
```javascript
// 使用环境变量
const SECRET_KEY = process.env.JWT_SECRET || 'default-secret-key'

// 或使用配置文件
import config from './config.js'
const SECRET_KEY = config.jwtSecret
```

### 4. CORS 配置过于宽松（高危）

**位置**：`server/server.js:18`

**问题描述**：
```javascript
app.use(cors())  // 允许所有来源访问
```

**风险等级**：🔴 高危

**影响**：
- 任何网站都可以调用你的 API
- 容易受到 CSRF 攻击
- 可能导致数据泄露

**修复建议**：
```javascript
app.use(cors({
  origin: process.env.ALLOWED_ORIGINS?.split(',') || ['http://localhost:5173'],
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization']
}))
```

## 🟡 中危漏洞

### 5. 缺少请求频率限制（中危）

**位置**：所有路由

**问题描述**：
没有实现 Rate Limiting，容易受到暴力破解和 DDoS 攻击

**风险等级**：🟡 中危

**影响**：
- 登录接口可以被暴力破解
- 验证码接口可以被刷爆
- 服务器资源可能被耗尽

**修复建议**：
```javascript
import rateLimit from 'express-rate-limit'

// 登录限流
const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15分钟
  max: 5, // 最多5次尝试
  message: '登录尝试次数过多，请15分钟后再试'
})

router.post('/loginin', loginLimiter, async (req, res) => {
  // ...
})

// 验证码限流
const emailLimiter = rateLimit({
  windowMs: 60 * 1000, // 1分钟
  max: 1, // 最多1次
  message: '发送验证码过于频繁，请1分钟后再试'
})

router.post('/sendemailcode', emailLimiter, async (req, res) => {
  // ...
})
```

### 6. 文件上传缺少严格验证（中危）

**位置**：`server/routers/file.js`

**问题描述**：
1. 只验证了文件扩展名，没有验证文件内容（Magic Number）
2. 文件大小限制不够严格（500个文件）
3. 没有限制单个用户的总存储空间

**风险等级**：🟡 中危

**影响**：
- 可以上传伪装的恶意文件
- 可能导致服务器存储空间耗尽
- 可能上传超大文件导致服务器崩溃

**修复建议**：
```javascript
import fileType from 'file-type'

// 验证文件真实类型
const validateFileType = async (filePath, allowedTypes) => {
  const type = await fileType.fromFile(filePath)
  if (!type || !allowedTypes.includes(type.mime)) {
    throw new Error('文件类型不合法')
  }
}

// 限制用户总存储空间
const checkUserQuota = async (userId) => {
  const [result] = await pool.execute(
    'SELECT SUM(fileSize) as totalSize FROM files WHERE publisherId = ?',
    [userId]
  )
  const totalSize = result[0].totalSize || 0
  const maxQuota = 1024 * 1024 * 1024 // 1GB
  if (totalSize >= maxQuota) {
    throw new Error('存储空间已满')
  }
}
```

### 7. 路径遍历漏洞风险（中危）

**位置**：`server/routers/file.js:download`

**问题描述**：
虽然使用了 `path.basename()`，但仍需要额外验证

**风险等级**：🟡 中危

**当前代码**：
```javascript
const filename = path.basename(relativePath)
const filePath = path.join(uploadPath, filename)
```

**潜在风险**：
如果 `relativePath` 包含 `../` 等特殊字符，可能导致路径遍历

**修复建议**：
```javascript
// 额外验证
const filename = path.basename(relativePath)
if (filename.includes('..') || filename.includes('/') || filename.includes('\\')) {
  return res.status(400).json({ status: 'fail', message: '非法文件路径' })
}

// 验证文件是否在允许的目录内
const filePath = path.join(uploadPath, filename)
const realPath = fs.realpathSync(filePath)
if (!realPath.startsWith(uploadPath)) {
  return res.status(403).json({ status: 'fail', message: '禁止访问' })
}
```

### 8. 缺少 HTTPS（中危）

**位置**：整个应用

**问题描述**：
应用运行在 HTTP 上，没有使用 HTTPS

**风险等级**：🟡 中危

**影响**：
- Token 可能在传输过程中被窃取
- 密码可能被中间人攻击截获
- 用户数据可能被篡改

**修复建议**：
```javascript
import https from 'https'
import fs from 'fs'

const options = {
  key: fs.readFileSync('path/to/private-key.pem'),
  cert: fs.readFileSync('path/to/certificate.pem')
}

https.createServer(options, app).listen(443, () => {
  console.log('HTTPS Server running on port 443')
})

// 强制重定向 HTTP 到 HTTPS
app.use((req, res, next) => {
  if (!req.secure) {
    return res.redirect('https://' + req.headers.host + req.url)
  }
  next()
})
```

## 🟢 低危漏洞

### 9. 缺少输入长度限制（低危）

**位置**：多个接口

**问题描述**：
部分接口没有限制输入长度，可能导致数据库溢出或性能问题

**风险等级**：🟢 低危

**修复建议**：
```javascript
// 添加输入验证
const validateInput = (input, maxLength, fieldName) => {
  if (!input || typeof input !== 'string') {
    throw new Error(`${fieldName}不能为空`)
  }
  if (input.length > maxLength) {
    throw new Error(`${fieldName}长度不能超过${maxLength}个字符`)
  }
  return input.trim()
}

// 使用示例
router.post('/createGroup', checkAuth, async (req, res) => {
  try {
    const groupName = validateInput(req.body.groupName, 100, '小组名称')
    // ...
  } catch (error) {
    return res.status(400).json({ status: 'fail', message: error.message })
  }
})
```

## 📝 总结

项目在 **SQL 注入防护**、**密码加密**、**权限验证** 方面做得很好，但在 **敏感信息管理**、**CORS 配置**、**频率限制** 方面存在严重问题。

**关键问题**：
1. 数据库凭证、JWT 密钥、邮箱密码等敏感信息硬编码
2. CORS 配置过于宽松
3. 缺少请求频率限制
4. 没有使用 HTTPS

**建议**：
优先修复高危和严重漏洞，特别是敏感信息泄露问题。这些问题修复后，系统的安全性将大幅提升。
